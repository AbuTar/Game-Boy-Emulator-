The Game Boy uses a Sharp LR35902 CPU

# Registers

The Game Boy has 8 8-bit General Purpose Registers as well as 2 16-bit registers

# Flags

F Register bits: Z N H C 0 0 0 0  Bits 0-3 always set to 0
                 │ │ │ │
                 │ │ │ └─ Carry (C): Did the result overflow?
                 │ │ └─── Half-Carry (H): Did bits 0-3 overflow into bit 4?
                 │ └───── Subtraction (N): Was this a subtraction?
                 └─────── Zero (Z): Is the result zero?


For a  half-carry, you use an AND (&) mask with the value being passed into the function and 0xF:
E.g.    0001 & 1111 (0xF)
        = 1 0000 - this is greater than 15 and so bit 3 overflowed into bit 4
        Code example: set_flag(cpu, FLAG_H, ((cpu->A & 0xF) + (value & 0xF)) > 0xF);
         - essentially set_flag(cpu, FLAG_H, (val in A + val in B > 15))

# Gameboy Timer

The Gameboy has 4 timer-related memory addresses:
0xFF04 - DIV  (Divider Register)
0xFF05 - TIMA (Timer Counter)
0xFF06 - TMA  (Timer Modulo)
0xFF07 - TAC  (Timer Control)

And the CPU runs at a a clock rate of 4,194,304 Hz

DIV - is not it's own register, rather the upper 8 bits of the Internal Divider
As a result, DIV increments every 256 cycles
- Writing any value to DIV resets the internal divider to 0

TIMA - Programmable timer Counter
- Watches specific bits of the internal dividers and increments whenever it resets

TAC Register

TAC bits: 0 0 0 0 0 T E S S
│ │
│ └─ Clock Select Bit 0
└─── Clock Select Bit 1

Bit 2 = Timer Enable

Clock select bits (0–1) choose which divider bit TIMA watches:
TAC bits   Divider bit watched   Frequency

00         bit 9                 4096 Hz
01         bit 3                 262144 Hz
10         bit 5                 65536 Hz
11         bit 7                 16384 Hz
